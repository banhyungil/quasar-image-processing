# 화면 레이아웃

## 1. 사이드바

- 사이드바에는 카테고리별 필터함수 목록을 볼 수 있는 창
- - 필터함수 카드 구성: 아이콘, 텍스트, 옵션 선택 아이콘 버튼, 적용 버튼

## 2. 사이드바 - 옵션 창

- 각 필터함수 별로 필요한 옵션들이 있는 창
- 필터함수 옵션 버튼을 누르면 토글되서 옆 사이트바로 표시되도록 구현
- 필터함수 별 옵션 컴포넌트를 만들 계획

## 3. 메인: 적용할 필터함수를 노드 리스트로 표현 창

- 세로 아코디언 형태로, 노드리스트 섹션 + preset 목록 표시

### 가. 노드리스트 섹션

- 필터함수를 적용하면 노드 리스트에 순서대로 쌓임
- 노드 리스트 대로 단계 별로 원본 이미지에 필터함수를 적용
- 각 노드 이동가능하도록 구현 (drag & drop)
- 각 노드 적용 여부를 토글 버튼으로 구현
- 각 노드 별 해당 순서에 맞게 처리된 썸네일 이미지를 확인 버튼
- 노드리스트를 저장하여 preset 목록에 추가할 수 있음
- 저장시에는 preset 이름을 지정하여 저장할 수 있는 팝업을 띄운다.
- preset은 노드리스트를 한번에 불러오는 기능

### 나. preset 섹션

- 저장한 노드리스트를 preset형태로 보이게한다.
- preset 이름과 적용버튼을 표시하고, 적용 버튼을 누르면 노드리스트에 로드된다.

## 4. 메인: 원본이미지, 처리이미지 표시 창

- 메인 탭, flow 탭 구분

### 가. 메인 탭

- 2개의 카드를 세로 축으로 표시 -> 원본 이미지, 최종 처리이미지 표시

### 나. flow 탭

- 노드리스트 순으로 처리이미지를 세로로 나열
- 스크롤되도록 구현

# 기능

## 1. 전체기능

- 추가된 노드리스트에 대한 최종 연산처리는 버튼을 통해 명시적으로 처리
- 저장 버튼을 통해서 연산처리 가능

## 2. 노드리스트

- 각 노드에 썸네일확인 버튼에 호버시 이미지 표시
- 썸네일 갱신은 명시적으로 사용자가 직접 수행하도록 한다.
- 노드 리스트 및 파라미터 정보가 바뀌면 썸네일 갱신이 필요성을 CSS나 텍스트, 아이콘 등으로 표시
