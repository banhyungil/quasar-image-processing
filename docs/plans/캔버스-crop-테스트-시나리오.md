# 캔버스 Crop & 풀해상도 기능 테스트 시나리오

## 1. Crop 생성 & 목록 관리

- [ ] SourceNode의 crop 버튼 클릭 → CropDialog 열림
- [ ] CropDialog에서 Shift+드래그 → 영역 선택 → Crop 생성 → 사이드바 Crop 탭에 표시
- [ ] CropDialog에서 확대 후 ✂ 버튼 → 현재 뷰포트로 Crop 생성
- [ ] Crop 탭에서 crop 클릭 → SourceNode 이미지가 crop으로 교체 + 해상도 뱃지 변경
- [ ] Crop 탭에서 "원본 이미지" 클릭 → SourceNode 원본 복귀 + 해상도 뱃지 복원
- [ ] Crop 탭에서 crop 삭제 → 활성 crop이면 다음 crop 또는 원본으로 전환
- [ ] 원본 이미지 초기화 → Crop 목록도 함께 초기화

## 2. Crop + 필터 체인 연동

- [ ] Crop 선택 상태에서 필터 추가 → crop 이미지 기준으로 처리
- [ ] Crop 전환 시 전체 리프 노드 재처리 확인
- [ ] Crop 해제 시 원본 기준으로 전체 재처리 확인
- [ ] 활성 crop이 아닌 crop 삭제 → 재처리 안 함 확인

## 3. 풀해상도 토글

- [ ] 풀해상도 OFF → 썸네일 해상도로 처리 (기존 동작)
- [ ] 풀해상도 ON → 원본 해상도로 처리, 이미지 품질 향상 확인
- [ ] 풀해상도 ON + Crop 없이 → 경고 알림 표시
- [ ] 풀해상도 토글 시 전체 리프 노드 재처리 확인
- [ ] 풀해상도 ON → 무손실 WebP 인코딩 확인 (이미지 일그러짐 없음)

## 4. 중간 노드 숨기기

- [ ] 토글 OFF → 모든 노드 이미지 표시 (기존 동작)
- [ ] 토글 ON → 리프 노드만 이미지 표시, 중간 노드는 placeholder
- [ ] 토글 ON → 백엔드에 returnNodeIds로 리프만 전달 (네트워크 절약)
- [ ] 토글 전환 시 전체 재처리 확인

## 5. 노드 리사이즈

- [ ] FilterNode 우하단 핸들 드래그 → 노드 크기 변경
- [ ] SourceNode 우하단 핸들 드래그 → 노드 크기 변경
- [ ] 리사이즈 후 하위 노드 추가 → 부모 사이즈 상속
- [ ] 툴바 사이즈 입력 + Enter → 전체 노드 일괄 변경
- [ ] 툴바 사이즈 입력 + 적용 버튼 → 전체 노드 일괄 변경
- [ ] 노드 hover → 좌하단에 노드 크기 뱃지 표시

## 6. 다중 선택

- [ ] Shift+클릭 → 다중 노드 선택, 선택 CSS(파란 테두리) 표시
- [ ] 다중 선택 후 한 노드 리사이즈 → 선택된 모든 노드 동일 사이즈 적용
- [ ] 다중 선택 시 마지막 클릭 노드 기준으로 파람 패널 표시
- [ ] 캔버스 빈 영역 클릭 → 선택 해제

## 7. 해상도 표시

- [ ] SourceNode: 이미지 업로드 → 우하단 해상도 뱃지 표시 (예: 1920x1080)
- [ ] SourceNode: Crop 선택 → 해상도 뱃지가 crop 사이즈로 변경
- [ ] FilterNode: 처리 후 우하단에 해상도 + 실행시간 뱃지 표시
- [ ] FilterNode: 풀해상도 ON → 해상도가 원본/crop 사이즈와 동일

## 8. 레이아웃

- [ ] 노드 크기 변경 후 정렬 버튼 → dagre가 실제 크기 반영하여 겹침 없이 배치
- [ ] 다양한 크기의 노드 혼재 → 겹침 없이 정렬
