## 1. 아이디어

- ImgPrcPage.vue 에 적용
- 필터함수 패널에 커스텀함수목록을 표시

현재 필터 탭 구조에 "커스텀" 카테고리를 추가한다.


#### 기존 카테고리와의 차이점

| 항목        | 기존 필터                       | 커스텀 필터                                                        |
| ----------- | ------------------------------- | ------------------------------------------------------------------ |
| 데이터 소스 | `FN_OPTIONS_MAP` (상수)         | API 조회 (`customFiltersApi.getCustomFilters`)                     |
| 추가        | 불필요                          | "+" 버튼 → 코드 에디터 다이얼로그                                  |
| 수정        | 불필요                          | 수정 버튼 → 코드 에디터 다이얼로그                                 |
| 삭제        | 불필요                          | 삭제 버튼                                                          |
| 드래그      | 지원                            | 동일하게 지원                                                      |
| 노드 추가   | `addFilterNode(prcType, label)` | `addFilterNode('custom', filterName)` + `parameters.filterId` 설정 |

### 2.3 커스텀 필터 노드 추가 로직

커스텀 필터는 `prcType: 'custom'`으로 통합되므로, `filterId`를 parameters에 포함해야 한다.

```typescript
function addCustomFilterNode(cf: CustomFilter) {
  const nodeId = `filter-${Date.now()}`;
  const newNode = {
    id: nodeId,
    type: 'filter',
    data: {
      algorithmNm: 'custom' as PrcType,
      label: cf.nm,
      enabled: true,
      parameters: {
        filterId: cf.id,
        ...cf.params, // 기본 파라미터
      },
    },
  };
  // 기존 addFilterNode 로직과 동일하게 노드 추가 + 연결
}
```

#### 파라미터 정의 스키마

DB `t_custom_filter.params` 컬럼(JSONB)에 `ParamFieldDef` 형식으로 저장한다.
기존 필터의 `PARAM_FIELDS`와 동일한 구조를 사용하여 ParamPanel에서 동적 폼 생성이 가능하다.

```jsonc
// t_custom_filter.params 저장 형식
[
  {
    "key": "ksize",
    "label": "커널 크기",
    "type": "number",
    "default": 3,
    "min": 1,
    "max": 31,
    "step": 2,
  },
  {
    "key": "weight",
    "label": "가중치",
    "type": "number",
    "default": 1.5,
    "min": 0.1,
    "max": 10,
    "step": 0.1,
  },
]
```

- 코드 에디터에서 파라미터 정의 행을 동적으로 추가/삭제
- 각 행: key(코드에서 접근할 이름), label(UI 표시명), type(number/select), default, min, max, step
- 사용자 코드에서는 `params.get("ksize", 3)` 형태로 접근
- ParamPanel에서 기존 `PARAM_FIELDS[prcType]`와 동일하게 렌더링됨

### 2.7 에러 처리

서버에서 `AppError`로 반환하는 에러 코드를 활용한다.

```typescript
// 에러 코드별 사용자 메시지 매핑
const ERROR_MESSAGES: Record<string, string> = {
  CUSTOM_FILTER_EXEC_ERROR: '코드 실행 중 오류가 발생했습니다.',
  CUSTOM_FILTER_NO_RESULT: 'result 변수에 값을 할당해주세요.',
  CUSTOM_FILTER_INVALID_NDIM: '결과 이미지의 차원이 올바르지 않습니다. (2D 또는 3D)',
  CUSTOM_FILTER_INVALID_CHANNELS: '결과 이미지의 채널 수가 올바르지 않습니다. (1, 3, 4)',
  CUSTOM_FILTER_INVALID_DTYPE: '결과 이미지의 타입이 uint8이 아닙니다.',
  CUSTOM_FILTER_NOT_FOUND: '커스텀 필터를 찾을 수 없습니다.',
};

// axios interceptor 또는 catch 블록에서 처리
function handleApiError(error: AxiosError) {
  const data = error.response?.data as { code?: string; message?: string };
  const msg = ERROR_MESSAGES[data?.code ?? ''] ?? data?.message ?? '알 수 없는 오류';
  Notify.create({ type: 'negative', message: msg });
}
```
