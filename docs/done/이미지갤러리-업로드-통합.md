# [프론트엔드] 이미지 갤러리 다이얼로그 개선

## 배경
SourceNode의 기존 2개 버튼(파일 업로드 / 기존 이미지) 흐름은 유지한다.
ImageGalleryDialog에 업로드/검색/삭제/수정 기능을 추가하여 다이얼로그 하나에서 이미지 관리를 처리한다.

## 변경 후 흐름
```
[SourceNode]
  ├─ "파일 업로드" → 기존 file input 프로세스 (변경 없음)
  └─ "기존 이미지" → ImageGalleryDialog
       ├─ 검색 (파일명, 용량 필터)
       ├─ 기존 이미지 그리드 (클릭하면 선택 완료)
       │    ├─ hover 시 삭제/수정 버튼 노출
       │    └─ 파일명 인라인 수정
       └─ "새 이미지 업로드" 영역
            → 파일 선택 → 파일명 입력 → [업로드] → 갤러리에 추가 & 자동 선택
```

## 구현 단계

### 1. 업로드 영역 (완료)
- 다이얼로그 상단에 "새 이미지 업로드" 점선 영역
- 파일 선택 후: 미리보기 + 파일명 입력 + 취소/업로드 버튼
- `imgPrcApi.uploadFile(file)` 호출 후 갤러리 리로드

### 2. 검색 기능
- **파일**: `src/components/dialog/ImageGalleryDialog.vue`
- 갤러리 그리드 위에 검색 영역 추가
  - 파일명 검색: `q-input` (debounce 적용)
  - 용량 필터: 선택적 (min/max 또는 드롭다운: 1MB 이하, 1~5MB, 5MB 이상 등)
- 검색 시 `imgPrcApi.getProcessingImage({ search, minSize, maxSize })` 호출
- **파일**: `src/apis/imgPrcApi.ts`
  - `getProcessingImage` 파라미터에 `search`, `minSize`, `maxSize` 추가
- **파일**: `src/types/imgPrcType.ts`
  - OpenAPI 타입 재생성 (`npm run gen:types`)

### 3. 삭제 기능
- **파일**: `src/components/dialog/ImageGalleryDialog.vue`
- 각 갤러리 아이템에 삭제 버튼 (hover 시 우상단 아이콘)
- 클릭 시 확인 다이얼로그 → `imgPrcApi.deleteFile(fileId)` 호출 → 목록에서 제거
- **파일**: `src/apis/imgPrcApi.ts`
  - `deleteFile(fileId: string)` 함수 추가 (`DELETE /image-processing/{fileId}`)

### 4. 파일명 수정 기능
- **파일**: `src/components/dialog/ImageGalleryDialog.vue`
- 갤러리 아이템에 수정 버튼 (hover 시 아이콘) 또는 파일명 더블클릭
- 인라인 편집 or 소형 다이얼로그로 파일명 입력
- `imgPrcApi.renameFile(fileId, newName)` 호출 → 목록 갱신
- **파일**: `src/apis/imgPrcApi.ts`
  - `renameFile(fileId: string, originNm: string)` 함수 추가 (`PATCH /image-processing/{fileId}`)

## 변경 범위
| 파일 | 변경 내용 |
|------|-----------|
| `ImageGalleryDialog.vue` | 검색/삭제/수정 UI 추가 |
| `imgPrcApi.ts` | `deleteFile`, `renameFile` 함수 추가, `getProcessingImage` 파라미터 확장 |
| `SourceNode.vue` | 변경 없음 |
| `ImgPrcPage.vue` | 변경 없음 |

## 재사용 함수
- `imgPrcApi.uploadFile(file)` — `POST /image-processing/upload`
- `imgPrcApi.getProcessingImage()` — `GET /image-processing`
