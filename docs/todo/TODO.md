# 이미지 확대 팝업 기능 확장 — 미니 에디터

## 개요
확대 팝업에서 임시로 필터를 적용하고 미리보기 후, 캔버스 노드에 반영하는 기능

## 뷰어 모드 (2단계)

### 모드 1: 전체 이미지 탐색
- 팝업 열림 시 기본 모드
- DZI 또는 원본 이미지를 OsdViewer로 표시
- 줌/팬으로 관심 영역 탐색

### 모드 2: Crop 비교 모드
- 필터 추가 시 자동 전환
- 현재 뷰포트 영역을 서버에 요청 → 노드 이미지 crop + 처리 crop 반환
- 오버레이 슬라이더(Before/After)로 비교

```
[모드 1: 전체 이미지 탐색]
┌──────────────────────────────────────────────────────────┐
│ 헤더                                               _ □ X │
├──────────────┬───────────────────────────────────────────┤
│ [필터 선택 ▼] │                                           │
│              │                                           │
│ (steps 없음) │        전체 이미지 (DZI/OsdViewer)          │
│              │        줌/팬으로 탐색                       │
│              │                                           │
├──────────────┴───────────────────────────────────────────┤
│                                                [닫기]    │
└──────────────────────────────────────────────────────────┘

         ↓ 필터 추가 시 자동 전환

[모드 2: Crop 비교 모드]
┌──────────────────────────────────────────────────────────┐
│ 헤더                                               _ □ X │
├──────────────┬───────────────────────────────────────────┤
│ [필터 선택 ▼] │                                           │
│              │            ◄━━━│━━━►                      │
├──────────────┤   노드 이미지 crop     │    처리 crop              │
│ 적용 Steps   │               │                           │
│ ▼ 1. blur    │    슬라이더로 경계 좌우 이동                 │

│   sigma: ━●  │                                           │
│ ▶ 2. canny ✕│                                           │
│ ▶ 3. sharpen✕│   [전체보기] ← 모드 1로 복귀                │
│              │                                           │

│                          [캔버스에 반영] [닫기]             │
└──────────────────────────────────────────────────────────┘
```

## 사이드 패널 구성 (위→아래)
```
┌──────────────┐
│ [필터 추가 ▼] │ ← FilterTreeSelect
├──────────────┤
│ Crop 목록     │
│ ┌──────────┐ │
│ │ crop1 ✓  │ ← 활성 crop (클릭 시 전환)
│ │ crop2  ✕ │
│ └──────────┘ │
│ [✂ 자르기]   │ ← 현재 뷰포트에서 새 crop 생성
├──────────────┤
│ 적용 Steps    │ ← 아코디언 (q-expansion-item)
│ ▼ 1. blur    │   펼침 → 파라미터 인라인 표시
│   kernel: ━● │   삭제(✕) 버튼
│ ▶ 2. canny ✕│
├──────────────┤
│ [캔버스에 반영]│
└──────────────┘
```

### Crop 목록 관리
- `cropList[]` — 여러 crop을 관리 (각각 cropId, viewport, nodeImageUrl)
- `activeCropId` — 현재 선택된 crop
- crop 선택 시 → 해당 crop의 nodeImage + processedImage를 비교/타임라인 모드에 적용
- [✂ 자르기] 버튼 → 탐색 모드에서 현재 뷰포트로 crop 생성 → 목록에 추가 → 자동 선택
- crop 삭제 시 → 서버 캐시도 삭제 (DELETE /preview/crop)

### Crop 프로세스 흐름
```
1. 팝업 열림 → 탐색 모드 (전체 이미지 OsdViewer)
2. 관심 영역으로 줌/팬 이동
3. [✂ 자르기] 클릭 → 현재 뷰포트로 crop 생성 → crop 목록에 추가
4. 필터 추가/조절 → 선택된 crop에 적용 → 비교/타임라인 표시
5. 다른 영역 보고 싶으면 → 탐색 모드 전환 → 이동 → 다시 자르기
6. crop 목록에서 이전 crop 선택 → 해당 crop의 결과 표시
7. "캔버스에 반영" → tempSteps를 노드로 추가
8. 팝업 닫힘 → 모든 crop 캐시 삭제
```

### 백엔드 API (신규)
```
POST /image-processing/preview
{
  fileId: string,
  steps: [...],              // 기존 노드 steps + tempSteps
  viewport: { x, y, w, h }, // crop 영역 (비율 또는 px)
  padding: number            // 경계 아티팩트 방지 (blur 등)
}
→ { nodeImage: blob, processed: blob }  // 노드 이미지 crop + 처리 crop
```

## 재사용 컴포넌트
- **FilterTreeSelect** — 필터 선택 드롭다운 (기존 컴포넌트)
- **ParamPanel 내부 필드 로직** — 슬라이더/입력/셀렉트 (PARAM_FIELDS 기반)
  - ParamPanel 자체는 레이아웃이 다르므로 필드 렌더링 부분만 공통 컴포넌트로 분리 검토

## 핵심 구조
- 팝업 내 **임시 steps 배열** 관리 (순차 적용, 추가/삭제)
- 필터 선택 → steps에 추가 → 아코디언 펼침 → 파라미터 조절
- 파라미터 변경 → debounce + abort로 서버 요청 → crop 이미지 갱신
- "캔버스에 반영" → 임시 steps를 해당 노드 하위에 필터 노드로 추가

## 뷰어 모드 3: Timeline (step별 결과 스크롤 비교)

### 개요
각 step의 중간 결과를 가로 스크롤로 나열하여 필터 적용 과정을 시각적으로 비교

### 레이아웃
```
┌──────────────────────────────────────────────────────────┐
│ 사이드 │ ← 가로 스크롤 →                                  │
│        │ [노드이미지] → [blur] → [canny] → [sharpen]     │
│        │  원본 crop     step1    step2     step3         │
│        │                                                 │
│        │  각 이미지 아래에 필터명 + 주요 파라미터 표시      │
└──────────────────────────────────────────────────────────┘
```

### 백엔드 변경
- `POST /preview/apply` 에 `includeIntermediates: boolean` 옵션 추가
- true일 때 각 step 중간 결과를 모두 반환
- 응답: `[{ stepIndex, label, image(blob/url) }, ...]`
- 또는 별도 엔드포인트 `POST /preview/apply-all` 분리

### 프론트 구현
- `TimelineViewer` 컴포넌트 신규 생성
- 가로 스크롤 컨테이너 + 각 이미지 카드
- 이미지 클릭 시 해당 step까지만 적용된 결과를 compare 모드로 전환 (선택)

### 모드 전환 정리
| 모드 | 설명 | 전환 트리거 |
|---|---|---|
| explore | 전체 이미지 탐색 (OsdViewer) | 기본, "전체보기" 버튼 |
| crop | crop 이미지 단독 표시 | ✂ 자르기 시 자동 |
| compare | Before/After 슬라이더 비교 | 필터 추가 시 자동, "비교 모드" 버튼 |
| timeline | step별 결과 가로 스크롤 비교 | "타임라인" 버튼 |

## Shift + 드래그 영역 선택 Crop
- 탐색 모드에서 `Shift + 드래그`로 이미지 위에 사각형 영역 선택
- 드래그 완료 시 해당 영역을 viewport로 crop 생성
- OsdViewer 위에 overlay로 선택 영역 사각형 표시 (반투명 파란색)
- Shift 누르는 동안 커서를 `crosshair`로 변경
- OSD의 팬/드래그와 충돌 방지: Shift 누른 동안 OSD 팬 비활성화

## 고려사항
- crop 경계에서 blur 등 필터의 경계 아티팩트 → padding으로 여유 영역 포함 후 결과에서 제거
- 줌 아웃 상태에서 crop 영역이 너무 크면 → 최대 crop 크기 제한 또는 리사이즈 후 처리
- 기존 batch-tree API와 별도 preview API 분리 (crop + 양쪽 결과 반환)
- 임시 상태는 팝업 닫으면 폐기, "반영" 시에만 캔버스에 추가
- "전체보기" 버튼으로 모드 1(DZI 탐색)과 모드 2(crop 비교) 자유 전환
