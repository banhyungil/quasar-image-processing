# 프론트 테스트 환경

## 설치된 라이브러리

### 1. Vitest — 테스트 러너

- Vite 네이티브 테스트 프레임워크
- Jest 호환 API (`describe`, `it`, `expect`, `vi`)
- HMR 지원으로 watch 모드에서 변경된 파일만 재실행
- `vitest.config.ts`에서 설정

```ts
// 기본 테스트
import { describe, it, expect } from 'vitest';

describe('utils', () => {
  it('adds numbers', () => {
    expect(1 + 2).toBe(3);
  });
});
```

```ts
// mock 함수
import { vi } from 'vitest';

const fn = vi.fn();
fn('hello');
expect(fn).toHaveBeenCalledWith('hello');
```

```ts
// 모듈 mock
vi.mock('src/apis/filesApi', () => ({
  getFiles: vi.fn().mockResolvedValue({ items: [], hasMore: false }),
}));
```

### 2. @vue/test-utils — Vue 컴포넌트 테스트

- Vue 공식 테스트 유틸리티
- `mount` / `shallowMount`로 컴포넌트 렌더링
- props, emit, slot, DOM 상호작용 테스트

```ts
import { mount } from '@vue/test-utils';
import MyComponent from 'src/components/MyComponent.vue';

// 마운트
const wrapper = mount(MyComponent, {
  props: { title: 'Hello' },
});

// DOM 확인
expect(wrapper.text()).toContain('Hello');

// 이벤트
await wrapper.find('button').trigger('click');
expect(wrapper.emitted('submit')).toBeTruthy();

// props 변경
await wrapper.setProps({ title: 'World' });
expect(wrapper.text()).toContain('World');
```

```ts
// shallowMount — 자식 컴포넌트를 stub으로 대체 (빠름)
const wrapper = shallowMount(ParentComponent);
```

### 3. happy-dom — 브라우저 환경 시뮬레이션

- 경량 DOM 구현체 (jsdom 대비 빠름)
- `vitest.config.ts`에서 `environment: 'happy-dom'` 설정
- 별도 코드 작성 불필요, 자동으로 `document`, `window` 등 제공

## 설정 파일

### vitest.config.ts

```ts
import { defineConfig } from 'vitest/config';
import vue from '@vitejs/plugin-vue';

export default defineConfig({
  plugins: [vue()],
  test: {
    environment: 'happy-dom',
    globals: true, // describe, it, expect 글로벌
    include: ['src/**/*.{test,spec}.{ts,tsx}'], // 테스트 파일 패턴
  },
  resolve: {
    alias: {
      src: fileURLToPath(new URL('./src', import.meta.url)),
    },
  },
});
```

### package.json 스크립트

```json
{
  "test": "vitest", // watch 모드
  "test:run": "vitest run" // 1회 실행
}
```

## 실행 방법

```bash
npm test          # watch 모드 (파일 변경 시 자동 재실행)
npm run test:run  # 1회 실행
```

- 테스트 대상 파일과 같은 디렉토리에 `.test.ts` 또는 `.spec.ts`로 생성
- 예: `src/composables/useCropManager.test.ts`

## 테스트 패턴 예시

### Composable 테스트

```ts
import { describe, it, expect, vi } from 'vitest';

vi.mock('src/apis/filesApi', () => ({
  createCrop: vi.fn().mockResolvedValue({
    cropId: 'test-id',
    nodeImageUrl: '/test.png',
    width: 100,
    height: 100,
  }),
  deleteCrop: vi.fn(),
}));

describe('useCropManager', () => {
  it('creates a crop', async () => {
    // composable 테스트 로직
  });
});
```

### API 함수 테스트 (axios mock)

```ts
import { describe, it, expect, vi } from 'vitest';
import { getFiles } from 'src/apis/filesApi';

vi.mock('src/boot/axios', () => ({
  api: {
    get: vi.fn().mockResolvedValue({ data: { items: [], hasMore: false } }),
    post: vi.fn(),
    delete: vi.fn(),
    patch: vi.fn(),
  },
}));

describe('filesApi', () => {
  it('getFiles returns file list', async () => {
    const result = await getFiles();
    expect(result.items).toEqual([]);
  });
});
```

### 전체 구조 요약

describe('기능 A') ← 테스트 스위트 (그룹)
│ beforeAll() ← 그룹 시작 시 1회
│ beforeEach() ← 매 테스트 전
│
├─ it('동작 1', expect...) ← 테스트 케이스
├─ it('동작 2', expect...) ← 테스트 케이스
│
│ afterEach() ← 매 테스트 후
│ afterAll() ← 그룹 종료 시 1회
│
└─ describe('하위 기능') ← 중첩 가능
└─ it('동작 3')

### MyNote

1. 함수설명
   1-1. describe: 테스트 그룹 (Test Suite)
   1-2. it: 개별 테스트 케이스 (test)
   1-3. expect: 단언 (Assertion)
   1-4. beforeEach / afterEach: 각 테스트 전후 실행
   1-5. beforeAll / afterAll: 그룹 전체에서 한번만 실행

2. Mocking

2-1. 함수 레벨

- 개별 함수 하나만 가짜로 교체

```ts
const fn = vi.fn(() => 42);
```

2-2. Module-level Mocking

- 모듈 경로에서 특정 객체를 임의의 객체로 교체하는 행위
- 테스트 용이하도록 특정 객체를 조작한다.

```ts
vi.mock('src/boot/axios', () => ({
  api: { post: vi.fn() },
}));
```

- 모듈 Mocking을 하면 테스트 시 해당 모듈을 사용한 모든 모듈에 동일하게 적용됨

### expect Matcher

| 매처                          | 의미                              |
| ----------------------------- | --------------------------------- |
| `.toBe(값)`                   | 원시값 일치 (`===`)               |
| `.toEqual(객체)`              | 객체/배열 깊은 비교               |
| `.toBeTruthy()`               | truthy 확인                       |
| `.toBeFalsy()`                | falsy 확인                        |
| `.toBeNull()`                 | null 확인                         |
| `.toBeUndefined()`            | undefined 확인                    |
| `.toBeDefined()`              | undefined가 아닌지 확인           |
| `.toBeGreaterThan(n)`         | n보다 큰지 확인                   |
| `.toBeLessThan(n)`            | n보다 작은지 확인                 |
| `.toContain(항목)`            | 배열/문자열에 포함 여부           |
| `.toHaveLength(n)`            | 배열/문자열 길이 확인             |
| `.toMatch(정규식)`            | 정규식 매칭 확인                  |
| `.toThrow()`                  | 에러 발생 확인                    |
| `.toHaveBeenCalled()`         | 함수 호출 여부 확인               |
| `.toHaveBeenCalledWith(인자)` | 특정 인자로 호출됐는지 확인       |
| `.toHaveBeenCalledTimes(n)`   | n번 호출됐는지 확인               |
| `.toHaveProperty(키, 값?)`    | 객체 속성 존재/값 확인            |
| `.toMatchObject(부분객체)`    | 객체가 부분적으로 일치하는지 확인 |
| `.resolves.toBe(값)`          | Promise 이행 값 확인              |
| `.rejects.toThrow()`          | Promise 거부 확인                 |
