# Claude Code Auto Memory 동작방식

## 전체 흐름

```
┌─────────────────────────────────────────────────────────────┐
│                      대화 시작                                │
│                         │                                    │
│                         v                                    │
│              ┌─────────────────────┐                         │
│              │  MEMORY.md 인덱스    │ ← 자동 로드             │
│              │  (목차 역할)         │                         │
│              └────────┬────────────┘                         │
│                       │                                      │
│                       v                                      │
│         관련 메모리 파일 참조하여 컨텍스트 구성                   │
└─────────────────────────────────────────────────────────────┘
                        │
                        v
┌─────────────────────────────────────────────────────────────┐
│                      대화 진행                                │
│                                                              │
│   사용자 메시지 ──────┬──────────────────────────────────>    │
│                      │                                       │
│                      v                                       │
│            ┌───────────────────┐                             │
│            │  메모리 트리거 감지  │                             │
│            └────────┬──────────┘                             │
│                     │                                        │
│          ┌──────────┼──────────┐                             │
│          v          v          v                             │
│     자동 감지    "기억해줘"   "잊어줘"                          │
│     (역할,선호   (수동 저장)  (수동 삭제)                       │
│      피드백 등)                                               │
└─────────────────────────────────────────────────────────────┘
                        │
                        v
┌─────────────────────────────────────────────────────────────┐
│                    저장 프로세스 (2단계)                        │
│                                                              │
│   Step 1: 개별 메모리 파일 생성/수정                            │
│   ┌─────────────────────────────────┐                        │
│   │  ---                            │                        │
│   │  name: 메모리 이름               │                        │
│   │  description: 설명               │                        │
│   │  type: user|feedback|project|   │                        │
│   │        reference                │                        │
│   │  ---                            │                        │
│   │  메모리 내용                     │                        │
│   └─────────────────────────────────┘                        │
│                     │                                        │
│   Step 2: MEMORY.md 인덱스에 포인터 추가                       │
│   ┌─────────────────────────────────┐                        │
│   │  - [user_role.md] - 사용자 역할  │                        │
│   │  - [feedback_style.md] - 스타일  │                        │
│   └─────────────────────────────────┘                        │
└─────────────────────────────────────────────────────────────┘


## 메모리 타입 분류

```
┌──────────────────────────────────────────────────────────┐
│                    Memory Types                          │
├──────────┬───────────────────────────────────────────────┤
│          │                                               │
│  user    │  사용자 역할, 지식 수준, 선호도                  │
│          │  예: "테스트 입문 단계", "백엔드 시니어"          │
│          │                                               │
├──────────┼───────────────────────────────────────────────┤
│          │                                               │
│ feedback │  작업 방식에 대한 피드백 (교정 + 긍정 확인)       │
│          │  예: "요약하지마", "한 PR로 묶는 게 좋았음"       │
│          │                                               │
├──────────┼───────────────────────────────────────────────┤
│          │                                               │
│ project  │  프로젝트 진행 상황, 목표, 의사결정 배경          │
│          │  예: "3/5부터 머지 프리즈", "인증 리팩토링 중"    │
│          │                                               │
├──────────┼───────────────────────────────────────────────┤
│          │                                               │
│reference │  외부 리소스 위치                               │
│          │  예: "버그 트래킹은 Linear INGEST 프로젝트"      │
│          │                                               │
└──────────┴───────────────────────────────────────────────┘
```


## 파일 구조

```
~/.claude/projects/<프로젝트경로>/memory/
│
├── MEMORY.md                ← 인덱스 (목차, 항상 자동 로드)
│
├── user_role.md             ← 사용자 정보
├── feedback_style.md        ← 응답 스타일 피드백
├── project_auth_refactor.md ← 프로젝트 진행 상황
└── reference_linear.md      ← 외부 리소스 참조
```


## 저장하지 않는 것

```
 X  코드 패턴, 아키텍처        → 코드를 직접 읽으면 알 수 있음
 X  git 히스토리               → git log로 확인 가능
 X  디버깅 해결책              → 코드와 커밋 메시지에 이미 있음
 X  CLAUDE.md에 있는 내용      → 이미 자동 로드됨
 X  현재 대화에서만 쓰는 임시 정보 → 대화 컨텍스트로 충분
```


## 다음 대화에서의 활용 흐름

```
새 대화 시작
     │
     v
MEMORY.md 로드 (인덱스)
     │
     v
사용자 질문 수신
     │
     v
관련 메모리 파일 참조 ──> "이 사용자는 테스트 입문 단계"
     │                    "요약하지 말 것"
     v                    "현재 인증 리팩토링 진행 중"
     │
     v
맥락을 반영한 응답 생성
```


## 사용자 관리 명령어

| 명령 | 동작 |
|---|---|
| "이거 기억해줘" | 즉시 메모리 파일 생성 |
| "기억하고 있는 거 뭐야?" | MEMORY.md 인덱스 + 관련 파일 조회 |
| "그 메모리 지워줘" | 해당 파일 삭제 + 인덱스에서 제거 |
| "메모리 무시해줘" | 해당 대화에서 메모리 참조 안 함 |
| VS Code에서 직접 편집 | memory 폴더를 열어서 md 파일 수정/삭제 |
